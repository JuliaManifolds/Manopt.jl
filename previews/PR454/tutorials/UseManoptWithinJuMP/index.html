Using Manopt.jl from within JuMP

Ronny Bergmann

In this tutorial we aim to illustrate how to use manifolds and the algorithms from Manopt.jl within the JuMP framework by implementing the get started tutorial again using JuMP combining it with for example their get started tutorial.

using JuMP, Manopt, Manifolds, ManifoldsBase, Random
using ManifoldDiff: grad_distance, prox_distance
Random.seed!(42);

First we generate the same data for the Riemannian center of mass as before

n = 100
œÉ = œÄ / 8
M = Sphere(2)
p0 = 1 / sqrt(2) * [1.0, 0.0, 1.0]
data = [exp(M, p0,  œÉ * rand(M; vector_at=p0)) for i in 1:n];

and we start by stating that our JuMP model is

model = Model(Manopt.JuMP_Optimizer)

Our cost and gradient are defined as

f(M, p) = sum(1 / (2 * n) * distance.(Ref(M), Ref(p), data) .^ 2)
grad_f(M, p) = sum(1 / n * grad_distance.(Ref(M), data, Ref(p)));

Variant I: Specify the Riemannian Objective

As the first variant, we can set the objective to an already implemented Riemannian function, using the ManifoldGradientObjective.

@objective(model, Min, ManifoldGradientObjective(f, grad_f))
ManifoldFirstOrderObjective{Manopt.AllocatingEvaluation, @NamedTuple{cost::typeof(Main.Notebook.f), gradient::typeof(Main.Notebook.grad_f)}}

We set the start point and look at the overall model to see that the solver that by default will be used is the gradient descent

set_start_value.(p, p0)

Then we can call the solver

optimize!(model)

and look at the solution

solution_summary(model)

Variant II: Specify cost in the embedding

As a second variant, we define the objective in the embedding, so here the $\mathbb R^3$, since then we can leave both the gradient computation and conversion from the Euclidean to Riemannian gradient to JuMP.

model2 = Model(Manopt.JuMP_Optimizer)
@variable(model2, p[i=1:3] in M, start = p0[i])
3-element Vector{VariableRef}:
 p[1]
 p[2]
 p[3]

We defined the cost function in the embedding in a form, that JuMP can perform AD with.

@objective(model2, Min, sum(sum((p - d) .^ 2) for d in data) / (2 * n))

One disadvantage of this formulation and leaving the AD to JuMP is, that the exponential map computations accumulate an error over time. We use a stabilized version to compensate for that by setting the retraction to the stabilized version of the exponential map, that for numerical stability performs a projection onto the manifold as well.

set_attribute(model2, "retraction_method", ManifoldsBase.StabilizedRetraction(ExponentialRetraction()))
set_start_value.(p, p0)
